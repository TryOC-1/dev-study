Command Line Interface¶

Airflow has a very rich command line interface allowing to perform many types of operation on a DAG, starting services and supporting development and testing.

usage: airflow [-h]
               {backfill,clear,run,test,task_state,webserver,scheduler,initdb,resetdb,list_dags,list_tasks,worker,serve_logs,flower,version}
               ...

Sub-commands:¶

backfill¶

Run subsections of a DAG for a specified date range

airflow backfill [-h] [-t TASK_REGEX] [-s START_DATE] [-e END_DATE] [-m] [-l]
                 [-a] [-i] [-sd SUBDIR]
                 dag_id

Positional Arguments¶

dag_id The id of the dag to run

Named Arguments¶

-t, --task_regex
 The regex to filter specific task_ids to backfill (optional)
-s, --start_date
 Override start_date YYYY-MM-DD
-e, --end_date Override end_date YYYY-MM-DD
-m, --mark_success
 

Mark jobs as succeeded without running them

Default: False

-l, --local

Run the task using the LocalExecutor

Default: False

-a, --include_adhoc
 

Include dags with the adhoc parameter.

Default: False

-i, --ignore_dependencies
 

Skip upstream tasks, run only the tasks matching the regexp. Only works in conjunction with task_regex

Default: False

-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

clear¶

Clear a set of task instance, as if they never ran

airflow clear [-h] [-t TASK_REGEX] [-s START_DATE] [-e END_DATE] [-u] [-f]
              [-r] [-d] [-sd SUBDIR]
              dag_id

Positional Arguments¶

dag_id The id of the dag to run

Named Arguments¶

-t, --task_regex
 The regex to filter specific task_ids to clear (optional)
-s, --start_date
 Overide start_date YYYY-MM-DD
-e, --end_date Overide end_date YYYY-MM-DD
-u, --upstream

Include upstream tasks

Default: False

-f, --only_failed
 

Only failed jobs

Default: False

-r, --only_running
 

Only running jobs

Default: False

-d, --downstream
 

Include downstream tasks

Default: False

-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

run¶

Run a single task instance

airflow run [-h] [-sd SUBDIR] [-m] [-f] [-l] [-i] [--ship_dag] [-p PICKLE]
            dag_id task_id execution_date

Positional Arguments¶

dag_id The id of the dag to run
task_id The task_id to run
execution_date The execution date to run

Named Arguments¶

-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

-m, --mark_success
 

Mark jobs as succeeded without running them

Default: False

-f, --force

Force a run regardless or previous success

Default: False

-l, --local

Runs the task locally, don’t use the executor

Default: False

-i, --ignore_dependencies
 

Ignore upstream and depends_on_past dependencies

Default: False

--ship_dag

Pickles (serializes) the DAG and ships it to the worker

Default: False

-p, --pickle Serialized pickle object of the entire dag (used internally)

test¶

Test a task instance. This will run a task without checking for dependencies or recording it’s state in the database.

airflow test [-h] [-sd SUBDIR] dag_id task_id execution_date

Positional Arguments¶

dag_id The id of the dag to run
task_id The task_id to run
execution_date The execution date to run

Named Arguments¶

-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

task_state¶

Get the status of a task instance.

airflow task_state [-h] [-sd SUBDIR] dag_id task_id execution_date

Positional Arguments¶

dag_id The id of the dag to check
task_id The task_id to check
execution_date The execution date to check

Named Arguments¶

-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

webserver¶

Start a Airflow webserver instance

airflow webserver [-h] [-p PORT] [-hn HOSTNAME] [-d]

Named Arguments¶

-p, --port

Set the port on which to run the web server

Default: 8080

-hn, --hostname
 

Set the hostname on which to run the web server

Default: “0.0.0.0”

-d, --debug

Use the server that ships with Flask in debug mode

Default: False

scheduler¶

Start a scheduler scheduler instance

airflow scheduler [-h] [-d DAG_ID] [-sd SUBDIR]

Named Arguments¶

-d, --dag_id The id of the dag to run
-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

initdb¶

Initialize the metadata database

airflow initdb [-h]

resetdb¶

Burn down and rebuild the metadata database

airflow resetdb [-h]

list_dags¶

List the DAGs

airflow list_dags [-h]

list_tasks¶

List the tasks whithin a DAG

airflow list_tasks [-h] [-t] [-sd SUBDIR] dag_id

Positional Arguments¶

dag_id The id of the dag

Named Arguments¶

-t, --tree

Tree view

Default: False

-sd, --subdir

File location or directory from which to look for the dag

Default: “/home/jone/airflow/dags”

worker¶

Start a Celery worker node

airflow worker [-h] [-q QUEUES]

Named Arguments¶

-q, --queues

Comma delimited list of queues to cater serve

Default: “default”

serve_logs¶

Serve logs generate by worker

airflow serve_logs [-h]

flower¶

Start a Celery Flower

airflow flower [-h] [-p PORT] [-a BROKER_API]

Named Arguments¶

-p, --port

The port

Default: “5555”

-a, --broker_api
 Broker api

version¶

Show version

airflow version [-h]
